# Как работает модель

## 1. Что мы предсказываем?

Представь, что ты буришь скважину в земле. Не вниз — а горизонтально, параллельно слоям породы. Твоя задача: понять, в каком именно геологическом слое находится бур в каждой точке.

**TVT** — это число, которое описывает геологическую позицию. Чем больше TVT, тем «глубже» в геологическом смысле находится точка скважины (хотя физическая глубина может быть другой).

У нас есть данные с реальных скважин: для КАЖДОЙ точки известен TVT (цель). На новых скважинах TVT неизвестен — его и нужно предсказать.

---

```
  1444  2983524  1069174   -9271   115.32       NaN    11252.00

```

### Данные для теста (test — 3 скважины)

То же самое, но **нет колонки TVT** (её и нужно предсказать):

```
   MD       X          Y          Z       GR     TVT_input
 11467  2983525  1069022   -9258   115.69   11236.02
 11468  2983525  1069022   -9259   115.58   11237.05
  ...
  1442  2983524  1069172   -9269   120.34   11250.00  ← последний известный TVT_input
  1443  2983524  1069173   -9270   118.91       NaN   ← надо предсказать TVT
  ...
```

### Что значат колонки

| Колонка | Значение | Есть в test? |
|---|---|---|
| **MD** | Глубина вдоль скважины (метры) | Да |
| **X, Y, Z** | Координаты в пространстве | Да |
| **GR** | Гамма-каротаж — измерение радиоактивности породы. Разные породы светятся по-разному. | Да |
| **TVT_input** | Известная геология до определённой точки | Да (до PS) |
| **TVT** | Цель — то, что предсказываем | **Только в train** |

## 3. Что такое Prediction Start (PS)?

Бур идёт вдоль скважины. Пока он в участке с известной геологией — TVT_input заполнен. Потом геология заканчивается — и TVT_input становится **пустым (NaN)**.

**Prediction Start (PS)** — это строка, где TVT_input впервые пропадает. Все строки ПОСЛЕ этой точки — их и нужно предсказать.

```
Строки 1..1442:  TVT_input известен     ← предсказывать НЕ надо
Строки 1443..5277: TVT_input = NaN       ← надо предсказать TVT
                                           ↑ Prediction Start
```

В train-скважинах это примерно **3,8 миллиона строк** после PS. В test — **14 151 строка** (именно столько ID в sample_submission.csv).

## 5. Наивный baseline (наш «пол» — RMSE 15.91)

Самый простой способ предсказания:

> «Раз TVT меняется медленно, давай просто скопируем последнее известное значение TVT_input на все строки после PS.»

Для скважины `000d7d20` последний TVT_input = 11250.00. Значит, все 3836 post-PS строк получают `tvt = 11250.00`.

**Результат:** RMSE = 15.91 (на train). Это нижняя граница — лучше этого «тупого» метода ML-модель должна быть сильнее. Наш LightGBM пока чуть хуже (20.58), потому что не копирует TVT_input напрямую, а учится на других признаках.

## 6. Как работает LightGBM baseline (наш рабочий вариант — RMSE 20.58)

### Что такое LightGBM?

Это алгоритм, который строит «дерево решений». Представь игру в 20 вопросов:

> «MD больше 12000?» → Да
> «GR больше 100?» → Нет
> «Координата Z меньше -9500?» → Да
> → **Предсказываем TVT = 11500**

LightGBM автоматически находит лучшие вопросы и порядок их задания. Строит не одно дерево, а тысячу (n_estimators=1000), каждое следующее исправляет ошибки предыдущего.

### Какие признаки (фичи) использует модель?

Модель смотрит на 10 чисел для каждой строки:

| Фича | Что это | Зачем нужно |
|---|---|---|
| **MD** | Глубина | Позиция вдоль скважины — главный ориентир |
| **X, Y, Z** | Координаты | Пространственное положение скважины |
| **GR** | Гамма-каротаж | Разные породы = разный GR. Помогает понять, в каком слое бур |
| **GR_is_missing** | 0 или 1 | Флаг: есть ли измерение GR. Иногда датчик не срабатывает |
| **MD_delta** | Разница MD с предыдущей строкой | Скорость движения бура. Резкие скачки = возможно, сменился слой |
| **MD_relative** | Позиция от 0 до 1 | Где мы в скважине: начало (0) или конец (1)? |
| **row_position** | Номер строки / всего строк | То же самое, но по номеру измерения |
| **last_tvt_input** | Последний известный TVT_input для этой скважины | **Самый важный признак!** Одно число на всю скважину. Даёт модели «якорь» — от какого значения отталкиваться |

Последний признак — ключевой. Без него модель предсказывает с RMSE 120. С ним — 20.58. Это улучшение в **5.8 раз**.

### Почему модель не копирует last_tvt_input тупо, как наивный baseline?

Потому что модель видит **контекст**: GR меняется → порода другая → TVT должен измениться. Если GR резко вырос — модель скорректирует предсказание вверх или вниз относительно last_tvt_input. Наивный baseline так не умеет — он просто ставит константу.

### Как проверяется качество (валидация)?

Нельзя просто перемешать все 3.8 миллиона строк и поделить на 5 частей — строки одной скважины похожи друг на друга, модель «подсмотрит» ответ.

Вместо этого мы делим **по скважинам**: 80% скважин (618 штук) — тренировка, 20% (155 штук) — проверка. И так 5 раз с разным разбиением (GroupKFold). Каждая скважина попадает ТОЛЬКО в одну часть — никакого подглядывания.

| Fold | RMSE |
|---|---|
| 1 | 18.62 |
| 2 | 17.06 |
| 3 | 17.00 |
| 4 | 27.30 |
| 5 | 22.90 |
| **Среднее** | **20.58 ± 3.99** |

Fold 4 сильно хуже остальных — значит есть скважины, которые модель предсказывает плохо. Это нормально: возможно, там сложная геология или мало данных.

### Почему на Kaggle (LB) результат 24.11, а не 20.58?

Kaggle проверяет на **других** скважинах (50% тестовых данных, которые мы не видим). Разница +3.5 между нашей проверкой и Kaggle — обычное дело. Это значит, что те скважины, которые Kaggle оставил себе для оценки, немного сложнее наших тренировочных.

## 8. Резюме

| Метод | Что делает | RMSE |
|---|---|---|
| Константа (наивный) | Копирует последний TVT_input | 15.91 |
| LightGBM (только геометрия) | MD, X, Y, Z, GR | 120.06 |
| **LightGBM + last_tvt_input** | **10 фич: геометрия + «якорь»** | **20.58 (CV) / 24.11 (LB)** |

Модель работает так:

1. Для каждой скважины знает «отправную точку» — последний известный TVT_input
2. Дальше смотрит на глубину (MD), координаты (X, Y, Z) и каротаж (GR)
3. Если GR меняется — значит порода изменилась — корректирует TVT
4. Если GR стабилен — TVT почти не меняется
5. Предсказывает для каждой post-PS строки своё значение TVT

Это не идеально (наивный baseline всё ещё лучше на 5 пунктов RMSE), но это **обучаемая модель**, которую можно улучшать: добавить rolling-фичи по GR, выравнивание с typewell, CatBoost/XGBoost, ансамбль.
